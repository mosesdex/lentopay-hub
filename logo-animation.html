Lentopay Logo Motion - Animation Specification v1.0

The logo doesn't just sit there. It settles, like money arriving.

Five sanctioned behaviours for the Lentopay mark - each demonstrated live below, each specified to the millisecond and the curve. Movement always means money moving: motion travels left → right or bottom → top, like a transaction completing.

Never-combine rules

  • Never run two logo animations simultaneously or back-to-back on one surface.
  • Never loop The Flow, The Handshake or The Spin-up. Only The Pulse loops.
  • Never play The Handshake for anything other than a genuinely completed action.
  • Never animate the logo during error states - the mark stays still when things go wrong.

Handoffs between states

  • Pulse → Handshake: finish the current pulse cycle, then morph. Never cut mid-breath.
  • Pulse → failure: stop the pulse at scale 1, fade the ring out in 150ms, show the error UI. No logo theatrics.
  • Spin-up → app: if load finishes early, jump to the settle phase - never stretch the splash.
¶

Performance & accessibility

Every sanctioned animation is built from compositor-friendly properties and degrades to stillness gracefully.

60fps, by construction

  • Animate transform and opacity only (plus stroke-dashoffset and fill on the SVG mark, which are cheap at logo scale).
  • No layout properties, no filters, no box-shadow animation. The Pulse glow is a scaled border, not a blur.
  • Apply will-change: transform only for the animation's lifetime; remove it after.
  • Test on mid-range Android over 3G-class connections - Lentopay's real-world floor.

prefers-reduced-motion

When the OS requests reduced motion, every animation on this very page collapses to the sanctioned fallback - a static logo fading in over 200ms, ease-out. Nothing draws, springs, sweeps or loops.

@media (prefers-reduced-motion: reduce) {
  .logo-anim * { animation: none; }
  .logo-anim { animation: fade 200ms
    cubic-bezier(0,0,0.2,1) both; }
}

Loading states still need feedback: keep a static badge plus a text label ("Processing…") - information never depends on motion.

Export & handoff

One source of truth per animation, exported per platform. Timings and curves in this document are normative - a Lottie that disagrees with this page is a bug.

Lottie / After Effects

  • Author in AE on a 480×480 comp (10× the 48-unit grid), 60fps, shape layers only - no rasters, no effects that Bodymovin can't serialize.
  • Draw-on via Trim Paths; The Handshake as a true path morph between the L and the check.
  • Export with Bodymovin; verify in the Lottie web player and on-device (iOS + Android) before release.
  • Colors must reference the canonical hexes exactly - no AE color drift.

Rive alternative

For product surfaces, a single Rive file with a state machine beats four separate Lotties:

  • States: idle → pulse → success / idle, driven by one txState input.
  • Handles the Pulse → Handshake handoff natively (finish-cycle-then-morph).
  • Smaller runtime than Lottie on mobile; keep the Lottie set as the marketing/web fallback.
